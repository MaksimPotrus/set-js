Chess board

Display a chessboard with given height dimensions and width, according to the principle:

*  *  *  *  *  *

  *  *  *  *  *  *  

*  *  *  *  *  *

  *  *  *  *  *  *

(12, 4, ‘*’)

Input parameters: length, width, symbol to display

Output: checkerboard string

  • length, width - natural number (integer, positive, != 0), [1, 64]
  • character is a string of length 1, not a space

Sorting triangles

Display triangles in descending order area.

Input parameters: array of objects triangle

Output: an ordered array of triangle names

Note:

  • Payment the area of the triangle should be produced by Heron's formula.
  • Each a triangle is defined by the names of the vertices and the lengths of its sides. 
  • Application should handle floating point input:

      { 

      vertices: ‘ABC’,

      a: 10,

      b: 20,

      c: 22.36

      },

      { 

      vertices: ‘YTO’,

      y: 10,

      o: 20,

      t: 28.36

      }

  • Name vertices must match the names of the sides
  • Output: [‘YTO’, ‘ABC’]
  • side triangle - (0, 65536)

Palindrome

Check if a number or part of it is a palindrome. For example, the number 1234437 is not a palindrome, but its part is a palindrome 3443. Numbers less than 10 do not count as palindrome.

Input parameters: number

Output: the palindrome extracted from the number, or 0 if the extraction failed.

out: the longest palindrome, or the left one among equal

Happy tickets

There are 2 ways to count lucky tickets:

  •   Simple — if it is printed on the ticket a six-digit number, and the sum of the first three digits is equal to the sum of the last three, then this ticket is considered lucky.
  •  Complicated — if the sum is even digits the ticket is equal to the sum of the odd digits of the ticket, then the ticket is considered happy.

Determine programmatically which variant of counting the lucky ones more tickets will be given at a given interval. 

Input parameters: an object context with fields min and max

Output: an object with information about the winning method and the number of lucky tickets for each way of counting.
